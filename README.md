# Adult Census Income: アンサンブル学習を用いた年収予測モデルの構築と最適化

## 1. プロジェクト概要
UCI Machine Learning Repositoryの「Adult Census Income」データセットを用い、年齢、教育水準、職業などの属性情報から、個人の年収が「5万ドルを超えるか（`>50K`）」を予測する二値分類モデルを構築しました。

本プロジェクトの目的は、現代の機械学習コンペティションや実務で主流となっている**アンサンブル学習（バギングとブースティング）の仕組みの違い**を理解し、特徴量エンジニアリングやハイパーパラメータチューニングを通じてモデルの限界性能を引き出すプロセスを実践することです。

## 2. 実施プロセス

### ① 欠損値の戦略的補完
カテゴリ変数の欠損に対し、データセットの特性に合わせた補完を実施しました。
- `workclass` (雇用形態) / `occupation` (職業): 「職業を回答していない」こと自体が特定の収入層を示唆する可能性があるため、単なる不明データではなく**「Unknown」という独立したカテゴリ**として扱い、予測のヒントとしました。
- `native.country` (出身国): 米国が全体の9割以上を占めるため、最頻値（`United-States`）で補完してノイズを低減しました。

### ② ドメイン知識に基づく特徴量エンジニアリング
モデルがより簡単に高収入層のパターンを学習できるよう、物理的な知見に基づく新しい特徴量を作成しました。
- **`Is_Married` (既婚フラグ)**: 7種類ある結婚ステータスのうち、世帯収入や経済的安定性に直結しやすい「配偶者がいる状態」を1、それ以外を0に統合。
- **`Capital_Net` / `Has_Capital_Activity`**: 資本収益と損失を合算して純損益を算出し、さらに「投資行動の有無」を示すフラグを作成して富裕層の特徴を際立たせました。
- **`Work_Intensity` (労働強度)**: 連続値である週の労働時間を、「Part-Time」「Full-Time」「Over-Time」の3つのライフスタイルカテゴリに分割しました。

### ③ アンサンブル学習モデルの比較と特徴量重要度の解釈
- **バギング (Random Forest)**: 独立した多数の決定木を並列に学習させ、多数決をとるベースラインモデルを構築。
- **ブースティング (LightGBM)**: 前の決定木が間違えた弱点を、次の決定木が直列的に修正していく強力なモデルを構築。
- **Feature Importanceの可視化**: どちらのモデルでも `age`（年齢）に次いで、自作した `Capital_Net` や `Is_Married` が上位にランクインしており、特徴量エンジニアリングの成功が裏付けられました。

### ④ GridSearchCVによるハイパーパラメータチューニング
- 軽度なデータ不均衡に対応するため、最適化のターゲット（Scoring）を正解率ではなく **F1-score** に設定。
- LightGBMのハイパーパラメータ（`learning_rate`, `max_depth`, `num_leaves`, `n_estimators`）を総当たりで探索し、過学習を防ぎつつ汎化性能を最大化する「黄金の組み合わせ」を特定しました。

## 3. 分析結果

| Model | Accuracy (正解率) | Precision (適合率) | Recall (再現率) | F1-score |
| :--- | :--- | :--- | :--- | :--- |
| Random Forest (Default) | 0.8485 | 0.7170 | 0.6122 | 0.6605 |
| **LightGBM (Tuned)** | **0.8723** | **0.7788** | **0.6556** | **0.7119** |

バギング（Random Forest）に対し、ブースティング（LightGBM）を適切にチューニングした結果、すべての指標で性能が上回りました。特に、モデルの総合力を示すF1-scoreを約5ポイント（0.66 → 0.71）引き上げることに成功しました。

## 4. 今回学んだ重要事項（Key Takeaways）

1. **欠損値は「情報」になり得る**
   - むやみに最頻値や平均値で埋めるのではなく、「欠損していること」自体が持つ傾向（Unknown層の収入の低さなど）をモデルに学習させることが有効な戦略になり得る。

2. **特徴量エンジニアリングの威力**
   - アルゴリズムの性能に依存するだけでなく、データ分析者がドメイン知識を用いて「予測に直結しそうなフラグやカテゴリ」を人間側で整理してあげることで、モデルの予測精度は飛躍的に向上する。

3. **アンサンブル学習の特性の使い分け**
   - **バギング (Random Forest)**: デフォルト設定でも過学習しにくく、安定したそこそこの精度が出せるため、最初のベースライン検証に最適。
   - **ブースティング (LightGBM)**: パラメータ設定に非常に敏感だが、適切に調整（チューニング）を施せばバギングを凌駕する圧倒的な精度を叩き出す。

4. **GridSearchCVによるシステマティックな最適化**
   - タスクの目的（今回は見逃しを防ぐためのF1-score重視）に合わせて評価指標を設定し、計算資源を使ってシステマティックに最適なハイパーパラメータを探索するパイプラインを習得した。
